### 4. 16staff_lookup_code.gs onOpen 修復

**問題**：員工（ant702212 / ysc810512）打開 16staff Sheet 只看到「🛠️ 訂貨工具」（只有樂樂匯入），看不到完整「🛠️ 開團管理」。

**根因**：onOpen 是「簡單觸發器」，對非試算表擁有者的個人 Gmail 帳號 `Session.getActiveUser().getEmail()` 回傳空字串（Google 隱私保護）→ email='' → 不在 ADMIN_EMAILS → 跑 else 分支。

**修法**：onOpen 取不到 email 時，預設當 admin 處理（員工 fallback）。實際權限由 Sheet 共用編輯權限控制（沒被 share 的人連 sheet 都打不開）。

```javascript
var email = '';
try { email = Session.getActiveUser().getEmail().toLowerCase(); } catch (e) {}
var isAdmin = email && ADMIN_EMAILS.some(...);
if (!email) isAdmin = true;  // ⚡ fallback
```

### 5. 補開 4 張 RT 退貨單（沖應收帳款）

**事故**：4 月發現 4 筆 sales_details 標記 `return_status='accepted'` 但 admin 沒對應開 RT 退貨單，造成應收帳款多算 **$2,285**。

**4 筆清單**：

| RT 單號（補開）| 原 SO | 店家 | 商品 | 沖帳金額 |
|---|---|---|---|---|
| RT20260407003 | SO20260407010 | 平鎮 | 澳洲M7和牛牛排 -30 | -$1,860 |
| RT20260415001 | SO20260415017 | 永和 | 豬鬃毛沐浴刷 -8 | -$200 |
| RT20260415002 | SO20260415019 | 經國 | 豬鬃毛沐浴刷 -4 | -$100 |
| RT20260415003 | SO20260415023 | 平鎮 | 豬鬃毛沐浴刷 -5 | -$125 |

**補開 SQL**（DO block + 雙重保護）：
- 保護 1：原 SO 已有 RT 不重複補（`EXISTS` 檢查 `ref_order_id`）
- 保護 2：原明細 `return_qty > 0` 才補
- 流水號自動算同日當前最大 +1
- 金額/數量/原因從 sales_details JOIN 帶（不手抄避免錯）
- BEGIN; ... ROLLBACK; 預演 → COMMIT

**驗證 SQL 跑完**：accepted 但無 RT 的單剩 0 筆 / $0 ✓

## 已知限制 / 設計取捨

1. **舊 RT 單沒填 `ref_order_id`**：4/20 之前 admin 在 branch_admin 開的 RT 單，`ref_order_id` 大多為 null（admin 沒填）。「退貨紀錄」分頁的「原銷貨單號」欄位會空白。**不補**（過去歷史，沖帳金額仍正確，補 ref 反而容易猜錯）。

2. **舊資料 `return_status='none'`** 是 portal 早期雜訊：sidebar UI 已過濾，不顯示「↳ x0 ( none )」這類無意義訊息。要徹底乾淨可跑：
   ```sql
   UPDATE sales_details SET return_status = NULL 
   WHERE LOWER(COALESCE(return_status, '')) IN ('none', 'null');
   ```
   現在不必跑（顯示已正常）。

3. **舊單英文值翻譯只在 sidebar 做**：DB 仍存英文（`accepted` / `return` 等），新 simple 系統用中文。**不改 DB**（避免影響 branch_admin 報表邏輯）。

4. **「所有進貨單」90 天 sliding window + 4/1 floor**：
   - 現在 ~ 6/30 過渡期：能看到完整 4 月舊單
   - 7/15 之後：4/16 之前的舊單會自然從清單消失（要查可用「🔍 查單號明細」直接輸入單號）

5. **「退貨紀錄」分頁查不到對應 RT 主表**：因為 `_fetchStoreOrdersForList_` 排除了 return type，sidebar 改走 `get_legacy_order_main` 補抓。
