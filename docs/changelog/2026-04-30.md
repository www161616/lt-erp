# 2026-04-30 — admin 對帳網站月結報表 (彙總 + 單店明細 + 月費 + 預留店轉店)

## 摘要

在 lt-erp-simple-sales（admin / 助理對帳網站）加月結報表功能：

1. **彙總表**（每店一列）：銷貨 / 退貨 / 月費 / 店轉店淨額 / 本月應收
2. **單店明細頁**（點店名進入）：日期 × 單號逐筆 + 小計 + 列印 + Excel
3. **固定月費**：每店一個固定值（預設 $5,000），可在彙總表 inline 編輯
4. **店轉店淨額預留欄位**：v1 固定 $0，等任務 6 店轉店做完後 v2 接入

第一版公式：
```
本月應收 = 銷貨 − 退貨 + 月費 + 店轉店淨額(=0)
```

純 read（除月費 UPSERT）；不動既有 simple_sales / sales_orders / 店家 Sheet 任何資料。

## 完成項目

### 1. 新建 DB schema (1 表 + 3 RPC)

**Migration 檔**：`docs/sql/migrations/2026-04-30_monthly_fees_and_store_detail.sql`

**新表 `store_monthly_fees`**：
- `store_name TEXT PK`、`monthly_fee NUMERIC DEFAULT 5000 CHECK >= 0`、`updated_at`、`updated_by`
- 啟用 RLS 但不加 policy → 預設 deny all 直接存取，只允許 SECURITY DEFINER RPC 通過
- 18 家店預先 INSERT $5,000（ON CONFLICT DO NOTHING，重跑安全）

**RPC 一覽**：

| RPC | 用途 |
|---|---|
| `update_store_monthly_fee_admin(secret, store_name, fee)` | UPSERT 月費，含 18 家店白名單防呆（拒絕「平鎭」等錯字）|
| `get_store_monthly_detail_admin(secret, store_name, year_month)` | 單店某月所有單（UNION ALL：simple / legacy_normal / legacy_return），依日期排序 |
| `get_monthly_summary_admin(secret, year_month)` | **改寫**：加 `monthly_fee` / `transfer_net` 欄位，net_amount 公式重算 |

**安全**：admin_secret OR JWT role 二擇一驗證（跟 simple_*_admin 同款）。

**白名單防呆**：update RPC 內檢查 store_name 是否在 18 家店名單，避免錯字 UPSERT 污染表。

## 已知限制 / 設計取捨

1. **店轉店淨額 v1 固定 $0**：等任務 6 店轉店做完後 v2 接入。UI 已預留欄位 + 灰字提示「v2 接入」，避免之後改大版面。

2. **月費是「每家店一個固定值」**：不分月。改一次套用所有月份。如果 admin 需要某月特例，目前只能臨時改、看完報表再改回。如果這個需求變多，v2 可改成 `(store_name, year_month)` 複合 PK。

3. **新系統退貨歸月**：用所屬銷貨單的 delivery_date（不另外追蹤退貨日期），原因是 simple_sales_details 沒有 received_at 時戳。實務上 admin 收到退貨即標 '已收到'，不會延月，影響不大。

4. **舊系統 RT 退貨歸月**：用 RT 自己的 order_date（與既有舊系統 MonthlyReport.html 行為一致）。

5. **暫存單不算銷貨**：is_draft=true 或 status='暫存' 的 simple_sales_orders 排除。確認送出後才入帳。

6. **退貨狀態判斷**：新系統只算 `simple_sales_details.return_status='已收到'`（admin 已實際收回貨）。'申請中' / '處理中' / '已拒絕' / '免退' 都不扣帳。最不會多扣。

7. **舊系統退貨**：直接用 RT 單 `ABS(grand_total)`（admin 開 RT 已等同確認沖帳）。

8. **客戶範圍**：只算 18 家分店 + 全民。樂鎂 / 恩蕾 / 央廚等其他客戶完全排除。

9. **月份範圍**：4 月起。1~3 月還在舊系統 MonthlyReport.html，不重複做。
