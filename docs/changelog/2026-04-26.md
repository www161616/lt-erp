## 部署交接清單

### 安裝（新 Spreadsheet 啟用此流程時）

1. **Apps Script 程式碼**：把 `google-apps-script/16staff_lookup_code.gs` 全文貼到 Apps Script 編輯器
2. **Script Properties**：⚙️ 專案設定 → 指令碼屬性 → 新增 `PICKER_SECRET`（值 = 從 Supabase 取得的 picker_secret）
3. **第一次跑**：F5 重整 Sheet → 看到「🛠️ 開團管理」選單 → 跑任意一個現有功能（會跳授權）→ 接受授權

### 撿貨員每日流程

1. 從月份分頁複製某結單日的列 → 新建分頁，命名 `撿貨表-YYYY-MM-DD`
2. 確認 18 家店欄位都在（含龍潭，順序：三峽中和文山四號永和忠順環球平鎮古華林口南平泰山萬華湖口經國松山全民龍潭）
3. 撿貨完，數量歸零代表沒撿、>0 代表撿到的數量
4. 選單 → 🛠️ 開團管理 → 📦 撿貨完成 → 一鍵建單 → 確認
5. 結果寫入「當日銷貨單成立表-YYYY-MM-DD」

### 常見錯誤與排解

| 錯誤訊息 | 原因 | 排解 |
|---------|------|------|
| Script Properties 未設定 PICKER_SECRET | 沒設 secret | 到 Apps Script 設定 |
| 您不在撿貨員白名單 | 該帳號不在 PICKER_EMAILS | 在 16staff_lookup_code.gs `PICKER_EMAILS` 加 email |
| 商品「XXX」在 products 表找不到 | 編號錯或產品未建檔 | 到 ERP ProductList 建商品或修正編號 |
| 商品「XXX」的 price_branch 未設定或 ≤ 0 | 該商品分店單價沒填 | 到 ERP ProductList 設 price_branch |
| 此撿貨表「...」已為「店名」建過單 | 重複建單（duplicate 偵測） | 正常行為，請確認原單號是否正確 |

### secret 管理

- 19 組 secret 都存在 Supabase `simple_sales_config` 表
- **絕不貼進對話 / git / Slack / 公開頁面**
- 換 secret 流程：`UPDATE simple_sales_config SET config_value = ... WHERE config_key = '...';` 然後到對應 Apps Script Properties 同步更新
- secret 洩漏處置：立即用 `INSERT ... ON CONFLICT DO UPDATE` + `gen_random_uuid()` 全量重產

### 任務 2.5：到貨制撿貨表流程（arrival-based picking sheet）

**動機**：原任務 2 假設「依結單日批次建撿貨表」，但實際撿貨員是依**廠商實際到貨的商品**撿（同一張撿貨表上可能有 4/1、4/6、4/10 不同結單日的商品）。撿完後撿貨員會列印拿紙本去現場撿，回來才填實撿。

**改動的檔案**：
- `google-apps-script/16staff_lookup_code.gs`（1515 → 1924 行）：加 onOpen 兩個新選單、`createTodayPickingSheet`、`_formatPickingSheetHeader_`、`showPickingSearchSidebar`、`searchProductsAcrossMonths`、`addPickedProductsToPickingSheet`、`_writeDiffSheet_`、`_computeDiff_`、`_summarizeDiff_`、`_parseExpectedFromNote_`；改寫 `_readPickingSheet_` 改用 cell value（實撿）+ getNote（應撿）；改 `createSalesOrdersFromPicking` 在建單後產生差額表
- `google-apps-script/PickingSearchSidebar.html`（新建，335 行）：搜尋框 + 卡片清單 + checkbox 勾選 + 批次加入
- `normalizeDate_` 加 Date 物件支援（修月份分頁的「結單日」欄是 Date 物件被 String() 變長字串的 bug）

**關鍵設計**：
- 撿貨表 23 欄（A 狀態 / B 結單日 / C 編號 / D 名稱 / E 合計 / F~W 18 家店）— 列印友善
- 各店欄位 cell value = 實撿（撿貨員回來填）；setNote `應撿:N` = 應撿（加商品時帶入訂購量）
- 列印時看到的數字是應撿（撿貨員拿著去現場撿），回來覆寫成實撿
- 建單後自動比對應撿 vs 實撿 → 寫「今日撿貨差額表-YYYY-MM-DD」分頁（顯示哪間店多/少）
- Sidebar 跨月份分頁搜尋（編號 OR 名稱），支援多筆勾選批次加入

**RPC 沒改**：v3.2 RPC 完全沒動，建單仍用實撿數量（cell value）當 qty。差額表由 Apps Script 算，不寫進 DB。

**安全強化**：
- `setNotes` 用 `''` 不用 `null`（Apps Script 行為一致）
- 差額表重跑時 `clearContent()` 第 2 列起再寫（避免累加）
- 搜尋過濾 `totalQty <= 0` 的空商品（避免員工誤勾）
- B 結單日欄、C 商品編號欄寫入前 `setNumberFormat('@')`（避免自動轉日期格式或長編號變科學記號）
