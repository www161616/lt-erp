# 2026-05-01 — admin 月結報表 v3：手動店轉店 + 調整金額

## 摘要

延續 4/30 月結報表（v2 含月費），補上**兩個手動調整欄位**：
1. **店轉店淨額** transfer_amount + 店轉店備註（每月每店可填，可正可負）
2. **調整金額** adjustment_amount + 調整備註（每月每店可填，可正可負）

公式從：
```
本月應收 = 銷貨 − 退貨 + 月費 + transfer_net(=0)
```
改成：
```
本月應收 = 銷貨 − 退貨 + 月費 + 店轉店淨額 + 調整金額
```

兩個欄位都用同一張表 + 同一支 RPC，前端兩個獨立 💾 按鈕避免會計按錯。

---

## 動機

3 月有些店家銷貨退回沒及時扣到，4 月對帳要補扣。原本只能改舊銷貨單（風險大、會改歷史資料）。新做法：在月結報表加「調整金額」欄記原因 + 金額，月結淨應收自動算出來，不動歷史資料。

順便把 v2 預留的「店轉店淨額」placeholder 改成手動輸入（任務 6 還沒接到自動店轉店資料來源前，先用手動填）。

## 完成項目

### 1. SQL（新檔）

**Migration**：`docs/sql/migrations/2026-05-01_monthly_adjustments.sql`

#### 新表 `store_monthly_adjustments`
```
store_name        TEXT
year_month        TEXT (CHECK YYYY-MM 格式)
transfer_amount   NUMERIC DEFAULT 0  (可正可負)
transfer_note     TEXT
adjustment_amount NUMERIC DEFAULT 0  (可正可負)
adjustment_note   TEXT
updated_at, updated_by
PRIMARY KEY (store_name, year_month)
```
- 一店一月一筆，兩種調整放同一張表
- ENABLE RLS 但無 policy → 預設 deny all 直接存取

#### 新 RPC `update_store_monthly_data_admin`
- 一支 RPC 處理 transfer 和 adjustment（用 `p_section='transfer'/'adjustment'` 區分）
- 18 家店白名單防呆（拒絕「平鎭」等錯字）
- section 值防呆
- 金額允許正負 + NULL 視為 0

#### 改 `get_monthly_summary_admin`
- 移除 v2 的 `transfer_net=0` 寫死欄位
- 新增 4 欄：`transfer_amount` / `transfer_note` / `adjustment_amount` / `adjustment_note`（從 store_monthly_adjustments LEFT JOIN）
- 公式更新：`net = sales − returns + monthly_fee + transfer_amount + adjustment_amount`

- saveSection 後自動 `loadMonthlyReport()` 重新整理彙總表（淨應收即時重算）

### 3. UX 設計重點

| 設計 | 原因 |
|---|---|
| 兩組 💾 分開（店轉店 / 調整）| 業務直觀，不會以為「按一個按鈕全部一起存」 |
| 後端用同一支 RPC（用 section 區分）| 邏輯統一，未來改規則只動一處 |
| 金額允許正負 | 4 月平鎮 `-1985`（補扣 3 月退貨）、5 月某店 `+300`（加收某費用）都用得到 |
| 備註欄存「為什麼調整」 | 對帳追溯時看得到原因（「3 月銷貨退回未扣」「4/12 泰山轉平鎮」）|
| 空字串擋下 | 避免不小心清空輸入框 → 變 0 → 數字默默不對 |

## 已知限制 / 設計取捨

1. **店轉店仍是手動填**：店轉店 transfer_orders 等基礎資料還沒接，所以 admin 看完月底資料手動算淨額填進來。等任務 6 完成後可改為自動算（保留欄位設計即可，只要在 RPC 裡讀新表）。

2. **per (store, year_month) 一筆**：同店同月只能有一組店轉店 + 一組調整。實務上會計就是月底結算一次，足夠。

3. **金額無上限**：CHECK 沒限金額大小，admin 能填很大的數字（如 -1000000）。如果要防大數字打錯字，未來可加 CHECK 約束。

4. **改月份不會清掉舊月份的調整**：5 月查詢時只看 2026-05 的調整，4 月的不會跑進來。資料按 year_month 分隔得很乾淨。
